리액트는 복잡한 사용자 인터페이스를 쉽게 구축하기 위한 JavaScript 라이브러리이다. 
Single Page Application(html페이지가 1개)을 개발하는 데 주로 사용된다.
리액트는 가상의 dom tree(virtual dom)을 생성하고 컴포넌트의 상태나 속성이 변경되면 해당 컴포넌트를 렌더링한다.
컴포넌트란, 한가지 이상의 기능을 수행하는 UI를 모듈화 해놓은 것이다. 각각의 컴포넌트를 조립해서 페이지를 생성한다. 
즉, 리액트는 여러개의 컴포넌트로 구성되어 있어 유지보수에 좋고 가상 DOM을 사용하여 변경된 부분만 업데이트하므로 
실제 DOM 조작을 최소화하고 효율적인 렌더링이 가능하다

<작동 순서>
가상의 돔을 생성 → 컴포넌트의 상태가 변경되면 컴포넌트를 다시 렌더링 → 새로운 가상 돔 생성 → 
이전의 가상 돔과 새로운 가상 돔 비교 → 비교해서 변경된 부분만 실제 DOM에 업데이트

그리고 리액트는 CSR(client side Rendering)이라는 렌더링 방식을 사용하는데 CSR은 사용자가 
웹 사이트에 접속했을 때 사용자측에서 모든 파일을 다운로드하고 사용자 브라우저에서 렌더링하는 방식이다. 
즉, 사용자 측에서 화면을 그리기 때문에 client side Rendering이라 부른다.
CSR은 처음 접속했을 때 이후에는 모든 데이터를 클라이언트에서 처리하므로 검색 엔진 최적화(SEO)에 불리할 수 있고 
첫 렌더링 이후에 화면 전환이 빠르다. 그만큼 서버측 부담이 줄고 사용자 하드웨어에 의존하여 화면을 랜더한다. 
어떤 웹페이지를 만드는지에 따라 중요한 부분이 무엇인지 생각하고 만들면 좋을 것 같다.
